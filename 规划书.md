# TrackBite 项目开发规划书 (11天冲刺版)

**项目目标**：开发一个基于 Gemini 2.0 Flash 的 UI Navigator AI Agent，集成健康、日程与库存数据，实现自动化的膳食决策与买菜/外卖导航。
**团队配置**：2 名初级开发者 (Agent Beginner)
**总预估耗时**：120 - 160 小时

---

## 核心开发模块与耗时分布

### 1. 基础架构与环境搭建 (预估 15-20 小时)
*   **GCP 环境配置 (5h)**：设置 Google Cloud 项目、启用 Vertex AI API、配置 Cloud Run 部署环境。
*   **后端框架 (5h)**：搭建 FastAPI/Flask 基础 API，集成 Google GenAI SDK。
*   **前端基础 (10h)**：React/Next.js 基础页面，实现文件上传（小票/报告）与推荐列表展示。

### 2. 多模态数据解析引擎 (预估 30-40 小时)
*   **视觉 OCR 解析 (15h)**：编写 Prompt 让 Gemini 从购物小票/体检报告中提取结构化数据（食材、日期、健康建议）。
*   **数据集成 (10h)**：集成 Google Calendar API 同步日程，并结合健康建议生成膳食逻辑。
*   **库存管理 (10h)**：使用 Firestore 存储食材库存，并基于保质期常识实现过期预警逻辑。

### 3. UI Navigator 核心开发 (预估 45-60 小时) —— **重难点**
*   **自动化环境 (10h)**：配置 Playwright/Puppeteer 在服务器端运行。
*   **视觉 UI 导航 (30h)**：实现“截图 -> Gemini 识别 -> 执行动作”的闭环，让 Agent 能在买菜/外卖网页上搜索并加购。
*   **流程串联 (15h)**：打通从“决策建议”到“自动执行加购”的完整业务流。

### 4. 演示与比赛交付准备 (预估 20-30 小时) —— **评审关键**
*   **架构图与文档 (10h)**：绘制系统架构图，编写详细的 README 和部署脚本。
*   **演示视频制作 (15h)**：录制并剪辑 4 分钟内的 Demo 视频（包含真实 UI 导航演示与英文配音/字幕）。
*   **GCP 部署验证 (5h)**：确保后端在 Google Cloud 稳定运行并提供部署证明。

---

## 11天冲刺阶段划分与双人分工 (Parallel Workflow)

为了最大化开发效率，建议 Developer A 侧重于“大脑与逻辑”，Developer B 侧重于“双手与界面”。

### 阶段一：基础建设 (Day 1-2)
*   **Developer A (Backend & Logic)**
    *   配置 GCP 项目、启用 Vertex AI API
    *   搭建 FastAPI 基础后端框架
    *   集成 Google GenAI SDK (Gemini 2.0 Flash)
*   **Developer B (Frontend & Automation)**
    *   搭建 React/Next.js 基础前端页面
    *   配置 Playwright 自动化运行环境
    *   实现基础的网页截图与预览功能

### 阶段二：核心解析与导航攻关 (Day 3-5)
*   **Developer A (Backend & Logic)**
    *   编写购物小票与体检报告的 OCR 解析 Prompt
    *   集成 Google Calendar API 获取用户日程
    *   设计并实现 Firestore 数据库存储食材库存
*   **Developer B (Frontend & Automation)**
    *   **核心攻关**：实现“截图 -> Gemini 识别 -> 执行点击/输入”的 UI 导航闭环
    *   编写针对特定买菜网站的自动搜索与加购逻辑
    *   开发前端文件上传与解析结果展示界面

### 阶段三：智能决策与流程串联 (Day 6-8)
*   **Developer A (Backend & Logic)**
    *   开发智能决策引擎：综合健康、日程、库存生成膳食建议
    *   实现食材过期提醒与优惠信息匹配逻辑
    *   完善后端 API 接口供前端调用
*   **Developer B (Frontend & Automation)**
    *   将“决策建议”与“UI 自动化导航”进行完整串联
    *   优化 UI 导航的准确性，处理缺货、弹窗等异常情况
    *   美化前端交互体验，实现流畅的自动化演示效果

### 阶段四：集成、文档与演示 (Day 9-10)
*   **Developer A (Backend & Logic)**
    *   编写 README.md、部署脚本与本地运行指南
    *   绘制系统架构图 (Architecture Diagram)
    *   协助 Developer B 进行系统最终调试
*   **Developer B (Frontend & Automation)**
    *   **关键任务**：录制并剪辑 4 分钟内的 Demo 演示视频
    *   为视频添加英文配音或字幕（符合比赛要求）
    *   准备 Devpost 提交所需的文字与图片材料

### 阶段五：最终交付 (Day 11)
*   **全员协作**
    *   将后端部署至 Google Cloud Run 并完成最终验证
    *   对照比赛 Checklist 检查所有提交项（代码库、视频、文档）
    *   在 Devpost 官网完成最终提交
