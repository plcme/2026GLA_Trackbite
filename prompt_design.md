

| 推理维度       | 权重 | 来源                   | 推理思路 |

| **健康与医疗** | 高 | 医疗信息库 / 用户健康档案    | 根据用户的医疗背景（如过敏原、慢性病、营养需求）提供基础饮食限制和建议。 |
| **运动表现**   | 中 | 日历 (Calendar) / 健身应用 | 识别健身记录，针对性补充营养（如运动后补充电解质、蛋白质等）。 |
| **时间与精力** | 高 | 日历 (Calendar)      | 分析日程繁忙程度，推测用户除去必要休息有多久空闲的烹饪时间/（外食的）交通时间，以及精力 |
| **餐食渠道** | 高 | 日历 (Calendar) | 根据日程决定：晚归订购外卖；在家做饭则买菜；已有外食计划则不推荐。推理做饭可用时间与精力。 |
| **库存管理**   | 中 | 冰箱库存系统 （根据小票推理） | 监控食材有效期，优先推荐消耗即将过期的食材，减少浪费。 |
| **消费决策**   | 低 | 商家优惠信息 / 历史偏好 | 匹配用户喜好与当前折扣，在合适的时间提醒购买打折食材。 |
| **营养学管理** | 高 | 历史饮食记录 | 分析近期饮食记录，判断是否外食频率过高（推荐清淡、少油少盐饮食）；同时追踪营养素摄入状况，若检测到某类营养素（如维生素、膳食纤维、蛋白质等）长期摄入不足，则在本次推荐中优先补充该营养素的食物来源。 |
