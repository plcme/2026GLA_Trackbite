# TrackBite User Stories

## 1. 数据输入与解析 (Data Input & Parsing)
- **US1.1 购物小票识别**: 作为一名用户，我希望能够上传购物小票的照片，以便系统自动识别我购买的食材种类，数量和购买日期。
- **US1.2 体检报告/医嘱分析**: 作为一名关注健康的用户，我希望能够上传体检报告或医嘱（PDF/图片），以便系统了解我的饮食禁忌（如：少油少盐、避开海鲜）。
- **US1.3 日历同步**: 作为一名忙碌的上班族，我希望系统能关联我的 Google Calendar，以便根据我的日程（如：加班、健身、聚餐）来调整晚餐建议，

## 2. 智能决策与提醒 (Smart Decision & Notification)
- **US2.1 饭点决策助手**: 作为一名纠结“今晚吃什么”的用户，我希望系统能综合我的健康状况、现有食材和日程安排，为我推荐最合适的食谱。

## 3. UI Navigator 自动化 (UI Navigation & Automation)
- **US3.1 自动外卖/买菜导航**: 作为一名追求效率的用户，我希望系统能根据决策结果，自动在买菜或外卖网页/App 上搜索食材、选择商品并加入购物车。
- **US3.2 跨应用授权交互**: 作为一名用户，我希望授权系统操作我的买菜 App，让它像真人一样在屏幕上完成复杂的选购流程。

## 4. 比赛合规性 (Contest Compliance)
- **US4.1 多模态交互演示**: 作为一名参赛者，我希望系统能展示“视觉输入（小票/报告）”到“视觉操作（UI 导航）”的完整闭环，以符合 Gemini Live Agent Challenge 的评审标准。
- **US4.2 Google Cloud 部署**: 作为一名开发者，我希望系统的后端稳定运行在 Google Cloud 上，并能通过架构图清晰展示 Gemini 如何驱动整个流程。
